**본문(body)** 을 채워 이관을 완료한다.

## ⚠️ 핵심 발견 — 라이브 소스가 사라짐

출발지로 지정된 **꿈들.com** 을 점검한 결과, 더 이상 교육성장네트워크 꿈들의
사이트가 아니었다.

- 도메인: `꿈들.com` → punycode `xn--cw0by0h.com`
- 원본 서버 TLS 인증서: **`CN=www.jejutriathlon.com`**
- 페이지 `<title>`: **「제주특별자치도철인3종협회」**
- 모든 경로가 `index.php?mode=main` 으로 JS 리다이렉트되며, 기존 꿈들의
  게시판/글 데이터는 남아 있지 않음.

즉, 도메인이 다른 단체로 넘어가(또는 재사용되어) **원본 게시글 본문을
라이브에서 가져올 수 없는 상태**다.

## 시도한 방법과 결과

| 방법 | 결과 |
|------|------|
| WebFetch `https://꿈들.com` | HTTP 503 |
| curl(프록시 경유) | 원본 인증서 `www.jejutriathlon.com` 확인 — 공유호스팅 |
| curl 본문 확인 | 「제주특별자치도철인3종협회」 페이지로 리다이렉트 |
| archive.org / web.archive.org | 실행 환경 egress 정책으로 차단(403) |

## 본문을 확보하려면 (택1)

1. **직접 제공** — 각 글 본문 텍스트(한글/워드/텍스트 등)를 주시면
   `assets/posts.js`의 해당 `body`에 채워 커밋한다.
2. **백업/덤프 제공** — 기존 PHP 사이트의 DB 덤프·HTML 백업·캐시 파일을
   주시면 거기서 본문을 추출한다.
3. **아카이브 접근 허용** — 이 환경에서 `archive.org` egress가 열리면
   과거 스냅샷에서 복원을 시도한다. (현재는 차단됨)

## 현재 브랜치 상태
- 홈페이지 골격(`claude/kkumdeul-homepage-migration-x223ah`)을 기반으로 구성.
- `posts.js`의 5개 항목은 **본문 입력만 하면 즉시 사이트에 반영**되는 상태.
- 본문은 임의로 작성하지 않았음(실제 비영리단체 공지/결과보고이므로
  사실과 다른 내용을 지어내면 안 됨).
